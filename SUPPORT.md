# Support

Use the public GitHub repo for self-serve support.

- Bugs and reproducible CLI, Action, pack, or MCP issues: open an issue.
- Setup questions and MCP help: start a Q&A discussion.
- Connector ideas and workflow feedback: start an Ideas discussion.

Before posting, redact:

- API keys, tokens, passwords, cookies, and credentials
- customer names or private account data
- private company policies, exports, tickets, or chat logs
- generated company-brain artifacts that include confidential operating knowledge

For local debugging, include:

- install source, such as `github:Keyroler1/opscanon#v0` or `opscanon@0.1.0`
- operating system and Node version
- command run
- redacted error output
- which generated report or pack failed
